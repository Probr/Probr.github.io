Frequently Asked Questions

Which devices are supported?

Probr is designed to support devices that run a *NIX (Linux, Mac OS X, UNIX) operating system, operate a wireless interface that can be set to monitor mode, and also have access to the internet. The system was tested with:

  • ODROID C1 with Ubuntu
  • Raspberry Pi B with Raspbian
  • TP-Link TL-MR3020 with OpenWRT
  • Mac OS X
  • Ubuntu 14.04

However, Probr is not limited to these devices. Any device satisfying the following requirements can be used:

  • (ba)sh shell
  • wget (with https support)
  • tcpdump

Which databases are supported?

Out-of-the box, Probr supports the following databases as a storage for sniffed packets:

  • MongoDB
  • InfluxDB

If you want to use any other database for your use case, you can easily extend Probr to do so, just by adding a handler. You can take the handlers for the existing databases as an example (here ). Then, you need to add your handler to the config variable "PROBR_HANDLERS" in here.

Do I need to know how the WiFi protocol works, or how to write sniffing scripts?

No! That's the beauty of it. Probr comes with predefined command templates for sniffing WiFi probe requests and uploading them to Probr's backend database, from where you can start with your own analyses.
